## Project Parch and Posey by Pipeline_Pioneers (Circle 4)
This project aims to deepen our understanding on the use of Git, Linux and SQL commands while working in a team to analyse the Parch and Posey Database

## Dataset
The project dataset can be found [here](https://github.com/jdbarillas/parchposey/tree/master/data-raw)
The database contains 5 csv data models:
- web_events.csv
- sales_reps.csv
- region.csv
- orders.csv
- accounts.csv

## Parch and Posey Analysis 
In this repository, we explored the Parch and Posey Database to derive insights from their business. We answered several questions to explore the factors that can boost profitability and streamline operations. 
In summary, sales made directly (Direct channel) dominated at about $639.7M, while Facebook and Organic sales contributed significantly having $103M and $102M also contributed significantly. Sales grew steadily from 2013–2016, peaking in 2016 across all regions before dropping sharply in 2017.
More insights have been reported in our PowerPoint presentation. (see link here: [canva](https://www.canva.com/design/DAGyb07pg8I/S90OF9PMRQpw_WZ0Rlu1Lg/view) )


## Procedures
We started by generating questions to explore the datasets. Some of these questions included:
- Best performing sales rep
- Best performing product
- Regional yearly performance
- Top performing accounts
- Sales channels deriving highest revenue
- Monthly revenue trend

These questions were then assigned to different members of the group to answer using SQL queries.
An EDA file was created where each member contributed to this file with their solutions by merges and commits from their individual branch.
With the answers in place, a powerpoint presentation was designed using Canva for communicating the observations.
